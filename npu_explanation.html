1. NPU 동작 원리

무엇을 하는 모듈인가?

이 NPU는 손글씨 알파벳 인식기입니다. 28×28 픽셀 손글씨 이미지(784개 픽셀)를 입력받아, A~Z 26개 알파벳 중 어느 것인지 분류하는 완전 연결 신경망(Fully Connected Neural Network)을 FPGA 하드웨어로 구현한 것입니다.

신경망 구조

입력층: 784 뉴런 (28×28 픽셀, 1비트 이진값) ↓ W1[784×64] + b1[64] + ReLU 은닉층: 64 뉴런 (uint8, 0~255) ↓ W2[64×26] + b2[26] 출력층: 26 뉴런 (int32 스코어, 최댓값 인덱스 → 알파벳)

전체 처리 흐름

단계FSM 상태동작소요 클럭
1IDLE대기. start 신호 수신 시 진입—
2LOAD_CANVASBRAM 첫 번째 주소 Pre-fetch1
3CALC_L1784×64 MAC (64 hidden 노드 연산)~50,176
4CALC_L264×26 MAC (26 출력 노드 연산)~1,664
5ARGMAX26개 스코어 중 최댓값 인덱스 추출26
6DONEdone=1, result 확정 → IDLE 복귀1

핵심 하드웨어 자원

4개의 블록 ROM

  • weight_rom_l1: W1 [784×64 = 50,176개] × int8
  • bias_rom_l1: b1 [64개] × int32
  • weight_rom_l2: W2 [64×26 = 1,664개] × int8
  • bias_rom_l2: b2 [26개] × int32

내부 레지스터 / RAM

  • hidden_layer[0:63]: L1 결과 저장 (uint8 × 64)
  • l2_score_buf[0:25]: L2 스코어 저장 (int32 × 26)
  • l1_acc, l2_acc: 32비트 누적 가산기
  • max_score, result: Argmax 추적

외장 BRAM 인터페이스

캔버스 픽셀 데이터는 외부 Block Design의 BRAM Generator Port B에 저장됩니다. NPU는 읽기 전용으로 접근하며, BRAM의 동기식 읽기 특성(주소 입력 후 1클럭 뒤 데이터 출력)을 맞추기 위해 canvas_addrb_reg에서 1클럭 선행 주소를 제공합니다.

// BRAM 1클럭 레이턴시 보정: 실제 주소 출력은 1클럭 지연
always @(posedge clk) begin
    canvas_addrb <= canvas_addrb_reg;  // 1클럭 파이프라인
end
// canvas_doutb는 canvas_addrb가 설정된 다음 클럭에 유효

2. 신경망 수학 — 정수 양자화 구현

Layer 1 (입력층 → 은닉층)

수식: h[j] = ReLU(clamp( (Σ x[i] × W1[j,i] + b1[j]) >> SHIFT_L1, 0, 255 )) x[i] : 1비트 픽셀 → uint8로 변환 (1→255, 0→0) W1[j,i]: int8 가중치 (-128 ~ +127) b1[j] : int32 바이어스 SHIFT_L1 = 10 (고정소수점 스케일링 조정) 결과 h[j]: uint8 (0 ~ 255)

실수 부동소수점 연산 대신 정수 고정소수점(fixed-point)을 사용하여 곱셈기(DSP 블록)만으로 추론합니다. >> 10은 2¹⁰=1024로 나누는 것과 동일합니다.

// 1비트 픽셀 → uint8 (255 또는 0)
wire [7:0] pixel_uint8 = canvas_doutb ? 8'hFF : 8'h00;

// MAC 누적 (32비트 누적기, 오버플로우 방지)
l1_acc <= l1_acc + ($signed({1'b0, pixel_uint8}) * w1_data);

// 784번 루프 완료 시: 바이어스 + 시프트 + ReLU + Clamp
scaled_out = (l1_acc + b1_data) >>> 10;
if      (scaled_out < 0)   hidden_layer[hidden_cnt] <= 8'h00;  // ReLU
else if (scaled_out > 255) hidden_layer[hidden_cnt] <= 8'hFF;  // Clamp
else                        hidden_layer[hidden_cnt] <= scaled_out[7:0];

Layer 2 (은닉층 → 출력층)

수식: score[k] = Σ h[j] × W2[k,j] + b2[k] (ReLU 없음, 음수 스코어 허용) h[j] : uint8 은닉 뉴런 값 W2[k,j]: int8 가중치 b2[k] : int32 바이어스 score[k]: int32 (Argmax에서 최댓값 인덱스 추출) 
l2_acc <= l2_acc + ($signed({1'b0, hidden_layer[hidden_cnt]}) * w2_data);
// 64번 루프 완료 시: 바이어스만 더함 (시프트/ReLU 없음)
l2_score_buf[class_cnt] <= l2_acc + b2_data;

Argmax

result = argmax(score[0], score[1], ..., score[25]) = 가장 큰 L2 스코어를 가진 인덱스 (0=A, 1=B, ..., 25=Z) 
// max_score는 최솟값(0x80000000)으로 초기화
if (l2_score_buf[class_cnt] > max_score) begin
    max_score <= l2_score_buf[class_cnt];
    result    <= class_cnt;
end

ROM 주소 계산

ROM주소 계산식범위비고
weight_rom_l1hidden_cnt × 784 + pixel_cnt0~50175행 우선(row-major) 배열
bias_rom_l1hidden_cnt0~63은닉 노드 인덱스
weight_rom_l2class_cnt × 64 + hidden_cnt0~1663행 우선 배열
bias_rom_l2class_cnt0~25출력 클래스 인덱스
주의: 모든 ROM은 Block RAM으로 추론되어 1클럭 읽기 레이턴시가 있습니다. MAC 연산은 ROM 주소를 제시하고 다음 클럭에 데이터가 나오는 방식이므로, 카운터와 ROM 데이터 사이에 1클럭 파이프라인 오프셋이 내재합니다.

3. npu_ctrl.v — NPU 연산 FSM 코어

신경망 추론 연산 전체를 FSM으로 제어하는 핵심 모듈입니다. ROM 4개를 인스턴스화하고, MAC 연산과 Argmax를 순차적으로 실행합니다.

FSM 상태 전이도

CALC_L1 상태 상세 동작

CALC_L1: begin
    // MAC: 누적 가산. pixel_cnt는 0~783까지 순환, hidden_cnt는 0~63
    l1_acc <= l1_acc + ($signed({1'b0, pixel_uint8}) * w1_data);

    if (pixel_cnt < 783) begin
        pixel_cnt        <= pixel_cnt + 1;
        canvas_addrb_reg <= pixel_cnt + 1;  // 다음 픽셀 BRAM 주소 선행 제공
    end else begin
        // hidden 노드 1개 완성 → 스케일링 + ReLU + Clamp
        scaled_out = (l1_acc + b1_data) >>> 10;
        if      (scaled_out < 0)   hidden_layer[hidden_cnt] <= 8'h00;
        else if (scaled_out > 255) hidden_layer[hidden_cnt] <= 8'hFF;
        else                        hidden_layer[hidden_cnt] <= scaled_out[7:0];

        l1_acc    <= 0;
        pixel_cnt <= 0;
        canvas_addrb_reg <= 0;       // 다음 노드를 위해 BRAM 주소 초기화

        if (hidden_cnt < 63) hidden_cnt <= hidden_cnt + 1;
        else begin hidden_cnt <= 0; state <= CALC_L2; end
    end
end

4. npu_axi.v — AXI4-Lite 슬레이브 래퍼

MicroBlaze가 AXI4-Lite 버스로 NPU를 제어하고 결과를 읽는 인터페이스입니다. npu_ctrl을 내부에 인스턴스화합니다.

레지스터 맵

오프셋이름비트 필드R/W설명
0x00CTRL[0]=start triggerR/W1을 쓰면 NPU 시작. done_sticky 초기화.
0x04STATUS[0]=done_sticky, [1]=busyR onlydone은 인식 완료 후 sticky 유지. busy=1이면 연산 중.
0x08RESULT[4:0]=resultR only최종 알파벳 인덱스 (0=A, 25=Z). done=1 후 안정.

done_sticky 동작

NPU core의 npu_done은 1클럭 펄스입니다. AXI 래퍼는 이를 sticky 레지스터로 변환하여 MicroBlaze가 폴링(polling)으로 언제든지 완료 여부를 확인할 수 있게 합니다.

if (npu_done) begin
    done_sticky    <= 1'b1;         // done 펄스 캡처 → sticky
    result_latched <= npu_result;   // 결과도 함께 래치
end
// CTRL에 start=1을 쓰면 done_sticky 초기화 (새 추론 준비)
if (S_AXI_WDATA[0] == 1'b1) done_sticky <= 1'b0;

npu_start 펄스 생성

// 매 클럭 초기화
npu_start <= 1'b0;

if (S_AXI_AWVALID && S_AXI_WVALID && !axi_awready) begin
    if (S_AXI_AWADDR[3:2] == 2'b00 && S_AXI_WDATA[0]) begin
        npu_start   <= 1'b1;  // 1클럭 펄스
        done_sticky <= 1'b0;  // 이전 완료 클리어
    end
end

MicroBlaze 드라이버 사용 예 (의사코드)

// 추론 시작
Xil_Out32(NPU_BASE + 0x00, 0x1);   // CTRL.start = 1

// 완료 대기 (폴링)
while ((Xil_In32(NPU_BASE + 0x04) & 0x1) == 0) {}  // STATUS.done_sticky

// 결과 읽기
uint32_t result = Xil_In32(NPU_BASE + 0x08) & 0x1F;
xil_printf("Result: %c\n", 'A' + result);

5. tb_npu.v — NPU 기능 시뮬레이션 테스트벤치

AXI 버스 타이밍을 검증하면서 동시에 NPU 추론 완료 여부를 확인하는 테스트벤치입니다. npu_axi를 직접 인스턴스화하고, AXI 신호를 간단하게 연결하여 start 신호만으로 구동합니다.

가상 BRAM 시뮬레이션

// 테스트용 784픽셀 메모리 (5픽셀마다 1개 '그려진' 지그재그 패턴)
reg virtual_bram [0:783];
for(i = 0; i < 784; i = i + 1) begin
    if (i % 5 == 0) virtual_bram[i] = 1'b1;
    else           virtual_bram[i] = 1'b0;
end

// Block Design BRAM처럼 1클럭 지연으로 데이터 제공
always @(posedge clk) begin
    if (canvas_enb) canvas_doutb <= virtual_bram[canvas_addrb];
end

시나리오

시나리오 1 — start 트리거

start = 1'b1;  #10;   // 1클럭(10ns) 유지
start = 1'b0;

AXI AWVALID+WVALID가 동시에 1이 되므로, npu_axi 내부에서 write_fire 조건이 성립하여 NPU start 펄스가 생성됩니다.

시나리오 2 — busy 플래그 모니터링

wait(uut.npu_busy == 1'b1);
$display("[TB] NPU 가속기 내부 연산 작동 중...");

start 직후 FSM이 LOAD_CANVAS로 진입하면 busy=1이 됩니다.

시나리오 3 — done 및 결과 확인

wait(uut.npu_done == 1'b1);
$display("최종 추론 출력: %d (0=A, 1=B, ...)", uut.npu_result);

약 50,000 클럭 이후 done=1이 되면 result에 0~25 범위의 값이 출력됩니다. 패턴에 따라 특정 알파벳으로 분류됩니다.

시뮬레이션 소요 시간: 약 150~160μs (15만 클럭). 시뮬레이터에서 충분한 시뮬레이션 시간을 설정해야 합니다.

6. tb_npu_axi.v — AXI 버스 레지스터 통합 테스트벤치

AXI 트랜잭션 태스크(axi_write, axi_read)를 정의하고, AXI 버스로 NPU를 제어하는 전체 흐름을 검증합니다. ROM 파일도 직접 로드합니다.

ROM 강제 로드

// #1: 시뮬레이션 시작 직후 ROM 메모리에 학습된 가중치 강제 주입
$readmemh("training/exported/weights_l1.mem", dut.npu_core_inst.w1_rom_inst.mem);
$readmemh("training/exported/biases_l1.mem",  dut.npu_core_inst.b1_rom_inst.mem);

테스트 입력 패턴

// 대각선+격자 패턴 (특정 알파벳과 유사할 수 있음)
for (i = 0; i < 784; i = i + 1)
    canvas_mem[i] = ((i % 29) == 0) || ((i % 28) == 7);

폴링 기반 완료 대기

timeout = 0;
rd = 32'd0;
while (rd[0] == 1'b0 && timeout < 25000) begin
    axi_read(4'h4, rd);   // STATUS 레지스터 읽기 [0]=done
    timeout = timeout + 1;
end
if (timeout >= 25000) begin $display("FAIL: NPU done timeout"); $finish; end

결과 검증

axi_read(4'h8, rd);   // RESULT 레지스터 읽기
if (rd[4:0] > 5'd25) begin
    $display("FAIL: RESULT out of range");
    $finish;
end
$display("PASS: result=%0d (0=A, 25=Z)", rd[4:0]);

7. 예상 파형 설명

파형 1 — NPU 전체 추론 타임라인 (축약)

clk_‾_‾_‾_‾... (100MHz)
start──‾_──────────────────────────────
busy────‾‾‾‾‾‾‾‾...(~51867클럭)...‾‾__
done────────────────────────────────‾_
state[IDLE][LC][──CALC_L1──][CALC_L2][ARG][DONE][IDLE]
result──────────────────────────────[0~25]──

start 펄스 → LOAD_CANVAS(1클럭) → CALC_L1(~50,176클럭) → CALC_L2(~1,664클럭) → ARGMAX(26클럭) → DONE(1클럭) → IDLE. 총 약 51,868 클럭 ≈ 518μs @ 100MHz.

파형 2 — CALC_L1 내부 루프 (pixel_cnt 순환)

hidden_cnt[ 0 ]────────────────────────[ 1 ]────────
pixel_cnt[0][1][2][3]...[782][783]─[0][1][2]...
canvas_addrb[0][1][2][3]...[782][783][0][1][2]...
canvas_doutb──[b0][b1][b2]...[b782][b783][b0]...
l1_acc[0][a0][a1][a2]...[a783]─[0][a0]...
hidden_layer[0]────────────────────[h0값]──────────

pixel_cnt가 0→783 순환하면서 l1_acc에 누적. 783 완료 시 hidden_layer[hidden_cnt]에 ReLU+Clamp 결과 기록 후 hidden_cnt+1. canvas_addrb는 BRAM 1클럭 레이턴시 때문에 pixel_cnt보다 1 앞서서 제공됩니다.

파형 3 — BRAM 레이턴시 보정

clk__‾‾__‾‾__‾‾__‾‾
pixel_cnt[ N ][N+1][N+2]
canvas_addrb_reg[ N+1][N+2][N+3] ← 1클럭 선행
canvas_addrb──[ N+1][N+2][N+3] ← FF 출력
canvas_doutb─────[ B[N+1] ][ B[N+2] ] ← 1클럭 지연
pixel_uint8(N+1)──────────[ B[N+1]→0or255 ]

BRAM은 동기식 읽기(synchronous read)이므로 주소를 제시한 클럭의 다음 클럭에 데이터가 나옵니다. 이를 보정하기 위해 canvas_addrb_reg는 pixel_cnt보다 1 앞서 제공됩니다.

파형 4 — CALC_L2 내부 루프

class_cnt[ 0 (=A) ]────────────[ 1 (=B) ]──
hidden_cnt[0][1][2]...[63]──────[0][1]...
l2_acc[0][a0][a1]...[a63]─[0][a0]...
l2_score_buf[0]────────────[score_A]──────────

class_cnt가 0(A)~25(Z)를 순환하면서 각 알파벳의 점수를 계산합니다. 64번의 MAC 후 l2_score_buf에 저장합니다.

파형 5 — ARGMAX

class_cnt[0][1][2][3]...[25]
l2_score_buf[cnt][s0][s1][s2][s3]...[s25]
max_score[s0][s0 or s1][max...]──[FINAL_MAX]
result[0][0 or 1][...]──────[FINAL_IDX]

max_score는 초기값(0x80000000, 최솟값)에서 시작하여, 더 큰 점수가 나타날 때마다 max_score와 result를 갱신합니다. 26클럭 후 result에 최종 알파벳 인덱스가 확정됩니다.

파형 6 — AXI 쓰기/읽기 사이클 (tb_npu_axi)

S_AXI_ACLK__‾‾__‾‾__‾‾__‾‾
── WRITE CTRL=0x1 ──
AWVALID/WVALID────‾‾________────
AWREADY/WREADY──────‾‾______────
BVALID──────────‾‾____──
npu_start──────────‾_──────
── READ STATUS ──
ARVALID────‾‾________
ARREADY──────‾‾______
RVALID──────────‾‾__
RDATA──────────[done=1, busy=0]

파형 7 — done_sticky 캡처

npu_done (core)────────────────‾_────────
done_sticky─────────────────‾‾‾‾‾‾‾‾
result_latched─────────────────[0~25 값]
npu_start (next)────────────────────────‾_
done_sticky(cleared)───────────────────────────_

npu_done은 1클럭 펄스이지만, done_sticky는 MicroBlaze가 다음 start를 쓸 때까지 High를 유지합니다. 폴링 기반 소프트웨어에서 안전하게 읽을 수 있습니다.

최적화 가능성

현재 방식 (직렬 MAC)

매 클럭 1번의 MAC 연산. 단순하지만 ~518μs가 소요됩니다. DSP 블록 1개만 사용합니다.

병렬 MAC 확장 가능

여러 MAC 유닛을 병렬로 두면 수십 배 가속이 가능합니다. DSP 블록을 더 사용하는 대신 레이턴시를 1/N로 줄일 수 있습니다.
